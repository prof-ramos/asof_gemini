### Tailwind CSS Screen Breakpoint Customization

Source: https://v3.tailwindcss.com/docs/theme

This snippet illustrates how to customize the responsive breakpoints within the `theme.screens` object in `tailwind.config.js`. These custom breakpoints affect when utility classes with responsive prefixes become active.

```javascript
/** @type {import('tailwindcss').Config} */

  theme: {
    screens: {
      'sm': '640px',
      'md': '768px',
      'lg': '1024px',
      'xl': '1280px',
      '2xl': '1536px',
    }
  }
}
```

### JavaScript for Managing Dark Mode Preference

Source: https://v3.tailwindcss.com/docs/dark-mode

Provides a JavaScript snippet for managing dark mode preferences. It handles setting the 'dark' class on the `<html>` element based on `localStorage` and the system's `prefers-color-scheme` media query.

```javascript
// On page load or when changing themes, best to add inline in `head` to avoid FOUC
document.documentElement.classList.toggle(
  'dark',
  localStorage.theme === 'dark' || (!('theme' in localStorage) && window.matchMedia('(prefers-color-scheme: dark)').matches)
)

// Whenever the user explicitly chooses light mode
localStorage.theme = 'light'

// Whenever the user explicitly chooses dark mode
localStorage.theme = 'dark'

// Whenever the user explicitly chooses to respect the OS preference
localStorage.removeItem('theme')
```

### Manual Dark Mode Toggling with HTML Example

Source: https://v3.tailwindcss.com/docs/dark-mode

Illustrates how dark mode functions when using the 'selector' strategy. It shows an HTML structure where the background color changes based on the presence of the 'dark' class on the `<html>` element.

```html
<!-- Dark mode not enabled -->
<html>
<body>
  <!-- Will be white -->
  <div class="bg-white dark:bg-black">
    <!-- ... -->
  </div>
</body>
</html>

<!-- Dark mode enabled -->
<html class="dark">
<body>
  <!-- Will be black -->
  <div class="bg-white dark:bg-black">
    <!-- ... -->
  </div>
</body>
</html>
